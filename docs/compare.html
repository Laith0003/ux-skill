Honesty card

Where UI UX Pro Max is genuinely good

Their recommender architecture is sound. The 5-parallel-search pattern is the right shape for design system recommendation, and we adopted it directly. Their breadth of industry rules (161) was the bar to beat. They were first to ship the Python-engine architecture for AI-coding UX skills, and the category exists because of work like theirs.

Where they lose: no anti-pattern linter, no brand specs, no motion presets, no mandatory discovery, no component manifest. Those are the moats we built.

Honesty card

Where frontend-design is genuinely good

It's official Anthropic. Trust signal. It's also intentionally minimal — a single well-written skill prompt rather than a system. For users who want one trusted knob and don't need depth, it's the right pick.

Where it loses: no structured data, no linter, no brand specs, no cross-IDE, no chained commands. It's a single-shot taste skill, not an engine.

Honesty card

Where taste-prompt skills are good

Zero setup. One file. The prompt does its job for the first few generations. For a weekend project or a quick mock, they're more than enough.

Where they lose: no determinism. The same prompt produces different outputs every run because the rules live in LLM judgment, not regex. No CI gate is possible. No chained workflow.

Section 4 / 6 — the generators

vs. design-html / design-shotgun / image-to-code

Pure code-generation plugins. They produce HTML/JSX from a prompt or an image. They are good at the step ux-skill calls /ux-design, but they have no opinion on the recommendation lane before it or the lint lane after it.

Honesty card

Where generators are good

If you already have a clear brief and just want code now, a pure generator is faster than walking through a discovery flow. They're complementary tools — ux-skill's /ux-design command does the same job but with the recommendation lane attached.

Section 5 / 6 — the library plugins

The honest position

ux-skill is the brain. Other plugins are limbs.

If you want to decide what to build — style + palette + type + motion + components + brand language + anti-slop rules — ux-skill wins. If you want to execute a specific known-shape task — pull from Figma, fetch a shadcn card, generate from a screenshot — pick the specialist and use ux-skill as the recommendation layer above it.

That's why we built the 17-IDE installer: ux-skill runs inside Cursor, Windsurf, Aider, and the rest. Use it alongside, not instead of, the specialists.
