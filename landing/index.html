22 slash commands

A vocabulary for the design step.

Each command loads the manifests it needs, runs deterministically, and emits a structured payload your IDE understands.

/ux-recommend
The 5-parallel-search engine. Brief in, full stack out: palette, type, components, motion, anti-patterns to avoid.
$ ux recommend —industry fintech —audience pro —mode dark
/ux-lint
51 regex rules. Runs in CI. Three-equal-cards, display-font misuse, generic CTAs, the lot.
$ ux lint —threshold high src/
/ux-discover
10-field intake protocol. No improvisation. Forces the brief before generation.
$ ux discover
/ux-frame
Locks the page frame: nav, hero, sections, footer. Layout before pixels.
$ ux frame —page landing
/ux-system
Emits design tokens (CSS vars, Tailwind config, Style Dictionary) from a recommendation.
$ ux system —format css
/ux-design
The full one-shot generate. Discovery + frame + system + components in one pass.
$ ux design "loyalty wallet for MENA SMBs"
/ux-component
Single component generate with every state pre-specced. No more loading-spinner-only outputs.
$ ux component card —states loading,empty,error
/ux-motion
57 motion presets. Spring physics, magnetic hovers, opsz-axis settles.
$ ux motion —preset opsz-settle
/ux-copy
Writes microcopy without filler verbs. No Elevate, Unleash, Seamless.
$ ux copy hero —tone direct
/ux-critique
Reads your draft and flags AI fingerprints with the fix inline.
$ ux critique src/Hero.tsx
/ux-audit
Whole-surface audit. Lint + UX laws + a11y + perf in one report.
$ ux audit ./
/ux-a11y
WCAG checks at the source level. Contrast, focus rings, ARIA gaps.
$ ux a11y —level AA
/ux-fix
Auto-fix where the fix is deterministic. Drop the emoji, replace the gradient, rename the placeholder.
$ ux fix —apply
/ux-polish
The last 5 percent: tracking, leading, balance, optical adjustments.
$ ux polish src/Landing.tsx
/ux-dashboard
Dashboard-specific patterns. Bento, density, mono numerals, status semantics.
$ ux dashboard —density compact
/ux-research
Competitive scan via the research-synthesizer sub-agent. Reads the category before designing.
$ ux research "MENA loyalty"
/ux-stats
Manifest counts. Sanity-check the engine before you trust the recommendation.
$ ux stats
/ux-case-study
Builds a case-study page from a brief plus the generated output.
$ ux case-study —project dot-wallet
/ux-next
Suggests the next move based on what you just shipped.
$ ux next
/ux-init
Detect IDEs and install for all in one pass.
$ ux init
/ux-workshop
Interactive thinking partner for the brief stage. Hidden Profile reads included.
$ ux workshop
/ux-expert
Routes to the right sub-agent: frontend, motion, copy, research, design-system.
$ ux expert —route motion
The linter in action

51 regex rules. Zero tolerance for fingerprints.

The linter reads your source the way a reviewer reads a manuscript. Three-equal-cards, display-font misuse, lone emoji as icon — each gets caught at lint, with the fix in italics.

5 sub-agents

Specialists for the steps you'd rather not improvise.

Dispatch them via the Task tool. Each one loads only the manifests it needs — tight context, deterministic output.

frontend-engineer

Writes the code. React, Vue, Blade, Astro — with mandatory loading, empty, error states; image discipline; SEO baked in.

Task({
  subagent_type: "frontend-engineer",
  prompt: "Build the hero per discovery payload"
})

motion-engineer

Owns motion. Spring physics, transform and opacity only, magnetic hovers, opsz-axis variable-font animations. Reduced-motion compliant by default.

Task({
  subagent_type: "motion-engineer",
  prompt: "Add scroll-pinned settle to the stats section"
})

copy-writer

Writes microcopy. Direct, brief, unpretentious. Bans filler verbs, generic CTAs, fake testimonials. Voice match enforced.

Task({
  subagent_type: "copy-writer",
  prompt: "Hero copy for a MENA loyalty wallet"
})

research-synthesizer

Reads the category. Pulls competitors, finds the dominant patterns and the boring tropes, returns a positioning brief. Hidden Profile aware.

Task({
  subagent_type: "research-synthesizer",
  prompt: "Scan MENA loyalty incumbents"
})

design-system-architect

Owns the token layer. Color scales, type ramp, spacing, motion presets. Outputs CSS vars, Tailwind config, Style Dictionary — pick your format.

Task({
  subagent_type: "design-system-architect",
  prompt: "Tokenize the recommendation as CSS vars"
})
